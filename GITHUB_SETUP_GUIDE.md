# GitHub Setup Guide for BTP Project

This guide will help you push your BTP project to GitHub.

---

## Prerequisites

1. **GitHub Account**: Make sure you have a GitHub account
   - If not, create one at https://github.com/join

2. **Git Installed**: Verify git is installed
   ```bash
   git --version
   ```
   If not installed, download from https://git-scm.com/

3. **GitHub Authentication**: You'll need either:
   - Personal Access Token (recommended)
   - SSH Key

---

## Step-by-Step Instructions

### Step 1: Create a New Repository on GitHub

1. Go to https://github.com/new
2. Fill in the details:
   - **Repository name**: `BTP-IoT-Healthcare-Monitoring` (or your choice)
   - **Description**: `IoT-Based Healthcare Monitoring System using Machine Learning`
   - **Visibility**: Choose Public or Private
   - **DO NOT** initialize with README, .gitignore, or license (we already have these)
3. Click "Create repository"
4. **Copy the repository URL** (you'll need this in Step 3)

---

### Step 2: Initialize Git in Your Project

Open Terminal and navigate to your project:

```bash
cd /Users/harshitraj/BTP
```

Initialize git repository:

```bash
git init
```

Configure your git identity (if not already done):

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

---

### Step 3: Add Remote Repository

Add the GitHub repository as remote (replace URL with yours):

```bash
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
```

Example:
```bash
git remote add origin https://github.com/harshitraj/BTP-IoT-Healthcare-Monitoring.git
```

Verify remote was added:

```bash
git remote -v
```

---

### Step 4: Add Files to Git

Add all files (respects .gitignore):

```bash
git add .
```

Check what will be committed:

```bash
git status
```

You should see:
- ‚úÖ Source code (src/)
- ‚úÖ Documentation (reports/, docs/, *.md)
- ‚úÖ Configuration (requirements.txt, .gitignore)
- ‚úÖ Scripts (run_*.sh)
- ‚úÖ Metadata (preprocessing_metadata.json)
- ‚ùå Large data files (ignored)
- ‚ùå Models (ignored)
- ‚ùå Virtual environment (ignored)

---

### Step 5: Create Initial Commit

Commit your files:

```bash
git commit -m "Initial commit: BTP IoT Healthcare Monitoring System

- Data integration pipeline
- Comprehensive preprocessing framework
- MLP implementation (9-class posture classification)
- Complete documentation and reports
- Organized directory structure
- Ready for improvements (IMU integration planned)"
```

---

### Step 6: Push to GitHub

#### Option A: Using HTTPS (with Personal Access Token)

1. **Create a Personal Access Token** (if you don't have one):
   - Go to: https://github.com/settings/tokens
   - Click "Generate new token (classic)"
   - Give it a name: "BTP Project"
   - Select scopes: Check "repo" (full control of private repositories)
   - Click "Generate token"
   - **COPY THE TOKEN** (you won't see it again!)

2. **Push to GitHub**:
   ```bash
   git branch -M main
   git push -u origin main
   ```

3. **Enter credentials when prompted**:
   - Username: your GitHub username
   - Password: **paste your Personal Access Token** (not your GitHub password!)

#### Option B: Using SSH (if you have SSH keys set up)

```bash
git branch -M main
git push -u origin main
```

---

### Step 7: Verify Upload

1. Go to your GitHub repository URL
2. Refresh the page
3. You should see all your files!

## Troubleshooting

### Problem 1: "remote origin already exists"

**Solution:**
```bash
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
```

### Problem 2: "Authentication failed"

**Solutions:**
- Make sure you're using Personal Access Token, not your GitHub password
- Verify token has "repo" scope
- Check username is correct

### Problem 3: "Large files detected"

**Solution:**
- Check .gitignore is working: `git status`
- If large files are staged, unstage them: `git reset`
- Update .gitignore and try again

### Problem 4: "Permission denied (publickey)"

**Solution:**
- Use HTTPS instead of SSH
- Or set up SSH keys: https://docs.github.com/en/authentication/connecting-to-github-with-ssh

## Best Practices

### 1. Commit Messages

Use clear, descriptive commit messages:

**Good:**
```
‚úÖ "Add IMU data integration for improved accuracy"
‚úÖ "Fix preprocessing bug in blood pressure parsing"
‚úÖ "Update README with new results"
```

**Bad:**
```
‚ùå "Update"
‚ùå "Fix"
‚ùå "Changes"
```

### 2. Commit Frequency

- Commit after completing a logical unit of work
- Don't wait too long between commits
- Commit before major changes

### 3. Branch Usage (Optional)

For major features, use branches:

```bash
# Create feature branch
git checkout -b feature/imu-integration

# Make changes, commit

# Push branch
git push -u origin feature/imu-integration

# Create Pull Request on GitHub
# Merge when done
```
